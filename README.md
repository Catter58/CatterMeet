# CatterMeet

Веб-сервис для транскрибации и диаризации аудио и видео файлов (до 4 часов). Поддерживает русский и английский языки. Запускается в одном Docker-контейнере.

## Требования

- [Docker](https://docs.docker.com/get-docker/) версии 20+
- [Docker Compose](https://docs.docker.com/compose/install/) версии 2+

## Быстрый старт

```bash
git clone <url репозитория>
cd CatterMeet
docker compose up --build
```

После первого запуска откройте в браузере: **http://localhost:8000**

> **Первый запуск** займёт несколько минут — Docker скачивает образы и устанавливает Python-зависимости (~2 ГБ). Последующие запуски быстрее (кэш слоёв).

## Использование

1. **Загрузите файл** — перетащите аудио или видео в зону загрузки (или нажмите «Выберите файл»).
2. **Дождитесь обработки** — статус задачи обновляется автоматически каждые 5 секунд. Обработка идёт в фоне.
3. **Просмотрите результат** — нажмите **Открыть** рядом с завершённой задачей.
4. **На странице результата:**
   - Аудиоплеер для прослушивания файла.
   - Транскрипт, разделённый по спикерам с таймкодами.
   - Поиск по тексту — найденные слова подсвечиваются.
   - Клик на реплику — плеер автоматически переходит к нужному моменту.

## Поддерживаемые форматы

Любые форматы, которые обрабатывает **ffmpeg**:

- Аудио: MP3, WAV, M4A, AAC, OGG, FLAC и другие
- Видео: MP4, MKV, MOV, AVI, WebM и другие

## Технические ограничения

- Обработка строго **последовательная** (одна задача за раз) для контроля потребления RAM.
- Рекомендуемый объём RAM: **3–4 ГБ**.
- Используется только **CPU** (GPU не требуется).
- Для диаризации автоматически определяется до **2 спикеров**.

## Данные

Загруженные файлы и база данных хранятся в директории `./data/` на хосте (монтируется через Docker volumes). При остановке контейнера данные сохраняются.

```
data/
├── uploads/   # загруженные файлы
└── db/        # SQLite база данных
```

## Остановка

```bash
docker compose down
```

## Пересборка после изменений

```bash
docker compose up --build
```
