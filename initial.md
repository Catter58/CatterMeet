Роль: Ты — Senior Full-Stack и ML Engineer. Твоя задача — написать веб-сервис для транскрибации и диаризации длинных (до 4 часов) аудио и видео файлов с фокусом на русский и английский языки.

Критическое ограничение: Приложение должно работать строго в рамках 3-4 ГБ оперативной памяти (RAM) и запускаться внутри ОДНОГО Docker-контейнера. Никаких микросервисов, внешних брокеров сообщений (Redis/RabbitMQ) или тяжелых СУБД (PostgreSQL).

Стек технологий:

Backend: Python 3.11+, FastAPI, asyncio.Queue (для фоновых задач).

База данных: SQLite3 (с использованием FTS5 для полнотекстового поиска).

ML-модели: faster-whisper (модель: large-v3-turbo, квантование int8), silero-vad (ONNX версия), легковесная модель для эмбеддингов спикеров (например, speechbrain/spkrec-ecapa-voxcelex) + scikit-learn (KMeans).

Frontend: React (Vite), Tailwind CSS, shadcn/ui.

Системные зависимости: ffmpeg (для конвертации аудио).

Проект должен быть реализован строго по следующим этапам. Выполняй их последовательно. Перед переходом к следующему шагу убедись, что текущий работает.

Шаг 1: Инициализация и настройка окружения (Всё в одном)
Создай структуру монорепозитория: папки backend и frontend.

Напиши Dockerfile (multi-stage build):

Stage 1: Сборка Vite/React приложения.

Stage 2: Установка python:3.11-slim, установка ffmpeg через apt-get, копирование собранной статики фронтенда в папку backend/static, установка Python-зависимостей.

Настрой FastAPI так, чтобы он раздавал статические файлы фронтенда из папки static и обрабатывал API-маршруты по префиксу /api.

Шаг 2: База данных SQLite и FTS5
Создай модуль для работы с SQLite.

Создай таблицы:

tasks (id, filename, status, created_at, updated_at). Статусы: pending, processing, completed, failed.

transcripts (id, task_id, start_time, end_time, speaker, text).

Настрой виртуальную таблицу FTS5 для поля text, чтобы обеспечить быстрый полнотекстовый поиск по транскриптам конкретной задачи (task_id).

Шаг 3: ML-Пайплайн и жесткий контроль памяти (ЯДРО СИСТЕМЫ)
Напиши класс/модуль AudioProcessor. Критическое правило: в памяти одновременно может находиться только ОДНА ML-модель.

Конвертация: Полученный файл прогоняется через ffmpeg (через subprocess) в формат 16kHz, mono, .wav.

VAD: Загрузи silero-vad. Найди все отрезки с речью. Сразу после этого выполни del vad_model и gc.collect().

STT: Загрузи WhisperModel("large-v3-turbo", device="cpu", compute_type="int8") из faster-whisper. Выполни транскрибацию только по таймкодам, полученным от VAD. Выполни del whisper_model и gc.collect().

Диаризация: Загрузи легковесную модель для извлечения голосовых эмбеддингов. Извлеки эмбеддинги для каждого речевого сегмента. Выгрузи модель и вызови gc.collect(). Примени KMeans(n_clusters=2) к полученным векторам для разделения на спикеров.

Сохрани результаты (таймкоды, текст, номер спикера) в SQLite. Удали временные аудиофайлы.

Шаг 4: API и Внутренняя очередь (Backend)
Реализуй глобальную asyncio.Queue в FastAPI.

Создай фоновый воркер (background task), который читает из очереди. Воркер должен обрабатывать строго 1 файл за раз, чтобы не превысить лимит ОЗУ.

Реализуй эндпоинты:

POST /api/upload — загрузка файла, создание записи в БД, добавление в очередь. Возвращает task_id.

GET /api/status/{task_id} — поллинг статуса задачи.

GET /api/transcript/{task_id} — получение готового текста с диаризацией.

GET /api/search — поиск по FTS5.

Шаг 5: Frontend (Минимализм на shadcn/ui)
Инициализируй Vite + React + Tailwind + shadcn/ui.

Добавь компоненты shadcn: Button, Input, Table, Progress, ScrollArea, Card.

Реализуй 3 основных экрана/блока:

Зона Drag & Drop для загрузки файлов.

Таблица загруженных файлов с их статусами (реализуй поллинг статуса раз в 5 секунд).

Страница результатов: минималистичный аудиоплеер слева/сверху и колонка с транскриптом (разделенным по спикерам) справа/снизу.

Добавь строку поиска над транскриптом. При вводе текста отфильтровывай реплики или подсвечивай найденные слова. При клике на реплику — плеер должен перематываться на нужный таймкод.

Начни с Шага 1. Сгенерируй Dockerfile, requirements.txt и структуру папок, и запроси у меня подтверждение перед переходом к коду.
